# Cultural Reference Buffer

The cultural reference buffer converts public inspiration into original platform architecture.

It exists so the platform can learn from broad archetypes without copying, adapting, or depending on protected characters, brands, stories, visuals, dialogue, or fictional worlds.

## Purpose

```text
cultural inspiration
  -> neutral archetype
  -> original platform role
  -> publishable artifact
```

## Core Rule

Inspiration is allowed.

Adaptation is not allowed without permission from the rights holder.

## Allowed Metadata

The buffer may store neutral factual metadata:

- reference title
- release year
- public source URL
- source type
- observed archetype
- neutral interpretation
- canonical platform role
- IP status

## Disallowed Material

The buffer must not store:

- copied dialogue
- protected character identity as an agent identity
- logos or marks
- artwork or screenshots
- distinctive costumes or visual style
- detailed plot reconstruction as product canon
- implied endorsement or affiliation

## Buffer Record

```json
{
  "id": "reference:example",
  "title": "Example Reference",
  "releaseYear": 2000,
  "sourceType": "movie_reference",
  "observedArchetype": "watchful protector",
  "neutralInterpretation": "a role that protects a system without owning final authority",
  "canonicalRole": "Guardian",
  "ipStatus": "inspiration_only_no_adaptation"
}
```

## Canonical Platform Roles

- Guardian
- Anchor
- Tool Steward
- Authority Bridge
- Digital Twin Companion
- Operating Domain

## Publication Rule

Published artifacts must use original platform-owned role names.

Any third-party cultural reference must be marked as inspiration only, not license, ownership, affiliation, endorsement, or adaptation.
