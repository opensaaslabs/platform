# Agent Character IP Policy

No agent may take the form, shape, name, likeness, costume, identity, voice, logo, symbol, or visual style of a protected real-world character in any platform artifact.

## Core Rule

Agents must be original platform roles.

Agents may be inspired by general archetypes, but they must not imitate or embody copyrighted, trademarked, or otherwise protected characters.

## Not Allowed

Without explicit permission from the rights holder, agents must not use:

- protected character names
- protected character likeness
- costumes or distinctive visual appearance
- logos, emblems, marks, or symbols
- recognizable fictional-world identity
- copied dialogue
- character voice or catchphrases
- story-specific behaviors as product identity
- screenshots, artwork, posters, or stills
- implied endorsement or affiliation

## Allowed

The platform may use original role names and neutral archetypes, such as:

- Guardian
- Anchor
- Tool Steward
- Authority Bridge
- Digital Twin Companion
- Operating Domain

## Agent Design Rule

Every agent must have:

- an original name
- an original description
- an original visual identity
- a clear platform duty
- a non-infringing role definition
- no dependency on third-party fictional characters

## Cultural Reference Rule

A cultural reference may be used only as inspiration during ideation.

Before publication, it must be translated into original architecture language.

```text
reference inspiration
  -> neutral archetype
  -> original platform role
```

## Final Rule

No real-world artifact produced by the platform may present an agent as a protected character.

The platform owns its roles only when the roles are original.
